# Business Problem

Administrative workflows often rely on semi-structured PDF and Word documents. These documents contain operational data needed for reporting, auditability, process monitoring, and decision-making.

However, when the data is extracted manually into spreadsheets, the workflow becomes:

- slow;
- difficult to audit;
- prone to human error;
- inconsistent across document formats;
- hard to scale;
- weak for real-time or recurring reporting.

This project proposes an end-to-end pipeline that extracts, validates, stores, models, and visualizes administrative case data.

## Business questions

1. How many administrative cases are processed per month?
2. Which request types are most frequent?
3. Which faculties or programs concentrate the highest volume?
4. What is the average processing time?
5. Which records have missing or low-quality data?
6. Which observations appear most frequently?
7. Where are the main operational bottlenecks?

## Expected value

- Reduce manual data preparation.
- Improve traceability and data quality.
- Enable faster reporting.
- Support evidence-based process improvement.
- Create an auditable analytical layer for Power BI.
